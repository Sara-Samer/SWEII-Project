Register Use Case:

Use Case ID:

1

Use Case Name:

Register

Actors:

 User => all types of users (buyer, seller and admin)

Pre-conditions:

Have an email

Post-conditions:

An account is created for the user.

Flow of events:

User Action

System Action

The user enters the username

Optional according to the system:

The system validates the username

The user enters his email.

The system validates the email.

The user enters a password

The system validates the password

If data f the user is validated and there is no problem, the system stores the data of the user.

Exceptions:

User Action

System Action

The user enters an invalid username

The system notifies the user to re-enter this field.

The user enters an invalid email.

The system notifies the user to re-enter it.
